# Limitações Canônicas

Esta branch não deve reivindicar capacidades além do que for demonstrado por testes automatizados.

Limitações atuais:

- Não há certificação externa.
- Não há reprodução independente registrada.
- Claims de produção devem ser evitados.
- Claims comerciais, ROI e valuation ficam fora do README técnico.
- Claims neurocognitivos ficam fora da release canônica do Runtime.
- SIGKILL deve ser testado por controlador externo, não por CTRL+C.
- HMAC deve ser validado no fluxo real de mensagens antes de ser declarado integrado.
- Catch-Up por delta precisa ser comprovado por teste.
- Consenso formal não deve ser reivindicado sem protocolo e teste específicos.
